üåê Introduction to Computer Networking

Computer **networking** is the practice of connecting two or more computing devices (like computers, servers, printers, and smartphones) together to allow them to share resources and exchange data. In simple terms, it's how devices talk to each other.

What is Networking?

At its core, networking exists for two main reasons:

  • **Resource Sharing:** Sharing hardware (e.g., a single printer for an entire office) and software (e.g., shared files and applications).
  • **Communication:** Enabling users and applications to exchange information (e.g., sending an email, browsing a website, or video conferencing).

Real-World Explanation: Think of a network like a city's road system. Each device is a building, and the cables/wireless signals are the roads connecting them. Data are the cars traveling on these roads to reach their destination.

üó∫Ô∏è Types of Networks

Networks are generally categorized by their **geographical size** and **scale**.

1. Personal Area Network (PAN)

  • Description: The smallest and most personal type of network. It connects devices centered around a single person, typically within a range of about 10 meters (33 feet).
  • Examples: Connecting your **Bluetooth** headset to your phone, syncing a smart watch with a laptop, or connecting devices via a USB cable.

2. Local Area Network (LAN)

  • Description: A network that connects devices in a **small, defined geographical area**. This could be a home, a small office, or a single building. LANs are characterized by high data transfer rates.
  • Examples: Your **home Wi-Fi network**, or all the computers in a school computer lab connected together.
LAN Example (ASCII Diagram)
Client PC <-> Switch <-> Server

3. Metropolitan Area Network (MAN)

  • Description: A network that spans a **city or a large campus**. It's larger than a LAN but smaller than a WAN. MANs often connect multiple LANs together.
  • Examples: A city-wide Wi-Fi network, or the network connecting all branches of a bank within one metropolitan area.

4. Wide Area Network (WAN)

  • Description: A network that covers a **large geographical area**, often crossing state, national, or even continental boundaries. WANs use public networks (like the telephone system) or dedicated private lines.
  • Examples: The most prominent example is the **Internet** itself. Corporate networks connecting headquarters to remote branch offices across the country.

üï∏Ô∏è Network Topologies

**Topology** refers to the physical or logical arrangement of connected devices in a network.

1. Bus Topology

  • Arrangement: All devices are connected to a single central cable, called the **backbone** or segment.
  • Pros: Easy to install, less cabling needed.
  • Cons: A break in the central cable shuts down the entire network. Difficult to troubleshoot.

2. Star Topology

  • Arrangement: Every device is individually connected to a **central device**, typically a **switch** or **hub**.
  • Pros: Failure of one device/cable only affects that device. Easy to add new devices.
  • Cons: Requires more cable than Bus. If the central device fails, the entire network goes down.
Star Topology (ASCII Diagram)
PC1 <-> Switch <-> PC2
PC3 <-> Switch <-> PC4

3. Ring Topology

  • Arrangement: Devices are connected to form a single, **closed loop**. Data travels in one direction around the ring.
  • Pros: Data flows efficiently at high speed.
  • Cons: A single break in the cable or failure of one device can disrupt the entire network.

4. Mesh Topology

  • Arrangement: Every device is connected to **every other device** on the network.
  • Pros: **Highly fault-tolerant** (data has many alternative paths). Very robust.
  • Cons: Extremely expensive and complex to install and maintain due to the vast number of connections required.

üö¶ The OSI Model

The **Open Systems Interconnection (OSI)** model is a conceptual framework used to describe the functions of a networking or telecommunication system. It divides the communication process into **seven distinct layers**. [Image of the 7 layers of the OSI model]

Understanding the Layers (from Top to Bottom)

Layer Name Function / Purpose Example Protocols
7 **Application** Provides the interface between the application and the network. What the user sees. HTTP, FTP, SMTP, DNS
6 **Presentation** Handles data formatting, encryption, and compression. Ensures data is readable by the receiving application. JPEG, MPEG, TLS/SSL
5 **Session** Manages the beginning, maintenance, and ending of a connection (a "session") between two devices. NetBIOS, RPC
4 **Transport** Manages segment delivery and error checking. Responsible for reliable data transfer. TCP, UDP
3 **Network** Handles logical addressing (IP addresses) and routing of data packets across different networks. IP, ICMP
2 **Data Link** Handles physical addressing (MAC addresses) and controls access to the medium (e.g., Ethernet). Ethernet, ARP
1 **Physical** Transmits raw binary data (bits) over the communication medium (cables, airwaves). The actual hardware. Cables, NICs, Hubs

Analogy: Sending a Letter

  • Application (7): You write the letter (the data).
  • Presentation (6): You format the letter and choose the language.
  • Session (5): You decide if you're sending it certified or standard.
  • Transport (4): The postal service ensures the pages stay together and arrive in the correct order.
  • Network (3): The postal service adds the destination ZIP code (IP address) for routing.
  • Data Link (2): The post office uses the full street address (MAC address) for final delivery.
  • Physical (1): The letter is placed in the mailbox and physically transported.

üíª The TCP/IP Model

The **Transmission Control Protocol/Internet Protocol (TCP/IP)** model is a practical and widely used model that defines how data should be exchanged over the Internet. It is generally considered a simplified version of the OSI model, collapsing the seven layers into four or five.

TCP/IP Layers (Four-Layer Model)

TCP/IP Layer Corresponding OSI Layers Core Function Example Protocols
**Application** Application, Presentation, Session Provides services to the application software. **HTTP**, **DNS**, **FTP**
**Transport** Transport Manages end-to-end communication and reliability. **TCP**, **UDP**
**Internet (or Network)** Network Handles logical addressing and routing across networks. **IP**, ICMP
**Network Interface (or Link)** Data Link, Physical Handles the physical connection and data transmission medium. Ethernet, Wi-Fi, ARP

Key Protocols: TCP vs. UDP

  • **TCP (Transmission Control Protocol):** A **connection-oriented** and **reliable** protocol. It ensures that all data packets arrive at the destination and in the correct order.
    • Used for: Web browsing (HTTP/HTTPS), Email (SMTP, POP3, IMAP), File Transfer (FTP).
  • **UDP (User Datagram Protocol):** A **connectionless** and **unreliable** protocol. It sends data without first establishing a connection or ensuring receipt. It is faster because it has less overhead.
    • Used for: Live streaming, VoIP (Voice over IP), DNS lookups, online gaming.

üî¢ IP Addressing and Subnetting

An **Internet Protocol (IP) address** is a numerical label assigned to every device participating in a computer network that uses the Internet Protocol for communication. It serves two main functions: **host identification** and **location addressing**.

IPv4 (Internet Protocol Version 4)

  • Format: A 32-bit address represented in **dotted-decimal notation** (e.g., 192.168.1.10).
  • Structure: Divided into four octets (8-bit sections) separated by periods.
  • Total Addresses: Offers approximately $4.3$ billion unique addresses, which are rapidly depleting.

IPv4 Classes

The original system categorized IPv4 addresses into classes based on the first octet, primarily defining the division between the **Network** and **Host** portions of the address.

Class First Octet Range Default Subnet Mask Use Case
**A** 1 - 126 255.0.0.0 Large networks (few networks, many hosts)
**B** 128 - 191 255.255.0.0 Medium-sized networks
**C** 192 - 223 255.255.255.0 Small networks (many networks, few hosts)

Subnetting

**Subnetting** is the practice of dividing a single large network into smaller, more manageable sub-networks, or **subnets**.

  • Purpose:
    1. **Efficiency:** Reduces network congestion by localizing traffic.
    2. **Organization:** Makes network management easier.
    3. **Security:** Creates boundaries to control data flow.
  • How it Works: It borrows bits from the **host portion** of an IP address and uses those bits to create new **network IDs** (subnets). The **Subnet Mask** is used to determine which part of the IP address is the Network ID and which part is the Host ID.

Example: In the address 192.168.10.5 / 24

  • 192.168.10.5 is the **IP Address**.
  • /24 is the **CIDR notation** (Classless Inter-Domain Routing), which means the first 24 bits are the Network ID.
  • The Network ID is **192.168.10.0**.
  • The Host ID is **5**.

IPv6 (Internet Protocol Version 6)

  • Format: A 128-bit address represented in **hexadecimal notation** (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334).
  • Structure: Divided into eight 16-bit blocks (quartets) separated by colons.
  • Total Addresses: Offers a virtually inexhaustible supply of addresses (3.4 × 1038), solving the IPv4 depletion issue.

üõ†Ô∏è Network Devices: The Building Blocks

These are the primary hardware components that form a network.

Hub

  • Function: A basic network device that connects multiple devices. It operates at **Layer 1 (Physical)** of the OSI model.
  • How it Works: A Hub is a **dumb device**. When it receives a signal on one port, it simply **repeats** (broadcasts) the signal out to **all other ports**. This creates a lot of unnecessary traffic (network collisions).

Switch

  • Function: Connects multiple devices and is much smarter than a hub. It operates at **Layer 2 (Data Link)**.
  • How it Works: A Switch maintains a **MAC Address Table** (or Content Addressable Memory - CAM table). When data comes in, the switch reads the destination MAC address and forwards the data *only* to the port where the destination device is connected. This makes it efficient and reduces collisions. [Image of a switch connecting multiple computers]

Router

  • Function: Connects **different networks** together. It operates at **Layer 3 (Network)**.
  • How it Works: A Router uses **IP addresses** to determine the best path (route) for data packets to travel between networks (e.g., between your home network and the Internet). **Routers route; switches switch.**

Access Point (AP)

  • Function: Creates a **Wireless Local Area Network (WLAN)**.
  • How it Works: An AP connects to a wired network (like a switch) and converts the wired signal into a radio signal, allowing wireless devices (laptops, phones) to connect to the network.

üí¨ Essential Application Protocols

These protocols operate mostly at the **Application Layer (OSI Layer 7)** and provide services to the user.

Protocol Full Name Purpose Example
**DHCP** Dynamic Host Configuration Protocol Automatically assigns **IP addresses**, subnet masks, and gateway info to devices joining a network. When your phone joins a Wi-Fi, it gets an IP address from the DHCP server.
**DNS** Domain Name System Translates human-readable **Domain Names** (e.g., www.google.com) into computer-readable **IP Addresses** (e.g., 142.250.67.164). You type a name; DNS finds the address.
**FTP** File Transfer Protocol A standard protocol used to transfer files between a client and a server. Uploading website files to a web server.
**HTTP** Hypertext Transfer Protocol The foundation of data communication for the **World Wide Web**. Used for requesting and receiving web pages. Accessing any non-secure website.
**HTTPS** Hypertext Transfer Protocol **Secure** A secure version of HTTP. It uses **TLS/SSL encryption** to secure the connection, protecting sensitive data. Seeing the padlock icon in your browser.

üõ£Ô∏è Routing Basics

**Routing** is the process of selecting paths in a network along which to send network traffic. This is the primary function of a **Router (OSI Layer 3)**.

1. Static Routing

  • Description: The network administrator manually creates and enters every path into the router's **routing table**.
  • Pros: Very simple, low overhead, high security.
  • Cons: Not scalable. Requires manual changes if the network topology changes.

2. Dynamic Routing

  • Description: Routers automatically exchange routing information with other routers and adjust their routing tables based on network changes. This uses **Routing Protocols**.
  • Pros: Highly scalable, automatically adapts to network changes/failures.
  • Cons: More complex to configure, uses more router resources.

Dynamic Routing Protocols

Dynamic protocols are classified into two types:

Type Protocols Scope Description
**Interior Gateway Protocols (IGP)** **RIP**, **OSPF** Within an **Autonomous System (AS)** (e.g., a single company network). Used for routing *inside* a network.
**Exterior Gateway Protocols (EGP)** **BGP** **Between** Autonomous Systems (e.g., between two different Internet Service Providers). Used for routing traffic across the global Internet.
  • **RIP (Routing Information Protocol):** An older, distance-vector protocol that uses **hop count** (number of routers) as its metric.
  • **OSPF (Open Shortest Path First):** A widely used, **link-state** protocol. It builds a complete map of the network and uses the shortest path calculation, making it highly efficient.
  • **BGP (Border Gateway Protocol):** The primary protocol that runs the **Internet**. It determines the best path between ISPs.

üõ°Ô∏è Network Security and Management

Firewall

  • Function: A network security device (hardware or software) that monitors and controls incoming and outgoing network traffic based on predetermined **security rules**.
  • How it Works: It acts as a barrier, inspecting data packets based on criteria like source/destination **IP address** and **port number**.
Firewall Concept (ASCII Diagram)
Internet <-> Firewall <-> Internal Network

Network Address Translation (NAT)

  • Function: Translates a private (internal) IP address into a public (external) IP address and vice-versa.
  • Purpose: Allows multiple devices on a private network (like your home LAN) to share a **single public IP address** when connecting to the Internet. This conserves the limited pool of IPv4 public addresses.

Virtual Local Area Network (VLAN)

  • Function: A **logical group** of network devices that appear to be on the same LAN, even if they are geographically separated or connected to different physical switches.
  • Purpose:
    1. **Security:** Isolates departments (e.g., Finance from Marketing).
    2. **Performance:** Reduces the size of a **broadcast domain**, improving network speed.

Network Security Basics

The core principles of network security are:

  1. **Confidentiality:** Ensuring data is only accessible to authorized users (using encryption).
  2. **Integrity:** Ensuring the accuracy and completeness of data (using hashing).
  3. **Availability:** Ensuring authorized users have uninterrupted access to resources and data (using redundancy and robust systems).

Common threats include: **Malware**, **Denial-of-Service (DoS) attacks**, **Phishing**, and **Unauthorized Access**. Security measures include **access control lists**, **strong passwords**, and **encryption**.

üîß Network Troubleshooting Tools

These command-line tools are essential for any network administrator or user.

1. Ping (Packet Internet Groper)

  • Purpose: Tests **reachability** and measures the **round-trip time (latency)** for messages sent from the originating host to a destination computer.
  • Protocol Used: ICMP (Internet Control Message Protocol).
  • Example: ping 8.8.8.8 (checks connection to Google's DNS server).

2. Traceroute / Tracert

  • Purpose: Displays the **route** (path) that packets take across an IP network and measures transit delays. It reveals every router (hop) the packet passes through.
  • Example: traceroute www.amazon.com (shows the full journey to the Amazon web server).

3. Nslookup (Name Server Lookup)

  • Purpose: A network utility used to query the **Domain Name System (DNS)** to obtain domain name or IP address mapping or other DNS records.
  • Example: nslookup facebook.com (returns the IP address associated with that domain).
